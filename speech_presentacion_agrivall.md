# Speech resumido — Presentación final Agrivall (5–7 min)

## 1. Portada y objetivo (45–60 s)
Buenos días. Voy a presentar el proyecto final Agrivall, una web funcional y responsive orientada a una finca o negocio rural. La idea principal del proyecto es unir en una misma plataforma tres partes: la venta de frutas, la promoción y reserva de una casa rural, y un blog de noticias.

El objetivo del sprint final era entregar una web completa o parcialmente funcional, con una parte pública cuidada y una zona de administración. En nuestro caso, el proyecto se ha desarrollado con Laravel y Blade, usando CSS propio para el front y Bootstrap principalmente para el backoffice.

## 2. Fases del desarrollo (60–75 s)
El desarrollo empezó con la toma de requisitos a partir del enunciado. Ahí se definieron los bloques principales: productos y pedidos, reservas de casa rural y blog. Después pasamos a una fase de diseño previo, donde organizamos la estructura de páginas, rutas, modelos y base de datos.

A partir de esa planificación se desarrollaron las funcionalidades: catálogo, carrito, checkout, gestión de pedidos, reservas, posts, usuarios y roles. Finalmente, se hicieron pruebas de navegación y formularios, además de ajustes visuales para que la web funcionara correctamente en móvil, tablet y ordenador.

## 3. Funcionalidades implementadas (75–90 s)
La web tiene una parte pública para los clientes y una parte privada para administración. En la venta de frutas, el usuario puede consultar el catálogo, añadir productos al carrito y realizar un pedido sin pago real. En administración se pueden gestionar productos y revisar pedidos.

También está la parte de casa rural, con una página pública para mostrar la información y un sistema de reservas con estados. Por último, se incluye un blog de noticias, pensado para publicar contenido relacionado con cultivos, ecología, cursos o novedades de Agrivall.

Además, el proyecto incluye autenticación, registro de usuarios y un rol de administrador, lo que permite separar las acciones del cliente normal de las tareas de gestión.

## 4. Responsive: mobile first (60 s)
Una parte importante del proyecto ha sido hacer que la web sea responsive. Para ello, la idea principal es aplicar un enfoque mobile first: primero pensar cómo se verá en móvil y después ampliar el diseño para pantallas más grandes.

Esto implica usar correctamente el meta viewport, evitar diseños con anchos fijos y organizar el contenido en una sola columna en móvil. Luego, en tablet o escritorio, esas columnas pueden pasar a mostrarse en dos o tres columnas, por ejemplo en tarjetas de productos o secciones informativas.

## 5. Trucos responsive (60 s)
Algunos trucos prácticos para conseguirlo son: usar columnas fluidas, adaptar márgenes y paddings según el tamaño de pantalla, hacer que las imágenes sean fluidas para que no rompan el diseño, y revisar elementos conflictivos como tablas, menús o tarjetas.

También es importante no probar solo en una resolución. Lo ideal es revisar la web en varios tamaños y, si es posible, en un móvil real. Así se detectan problemas de navegación, textos demasiado largos o botones difíciles de pulsar.

## 6. Futuras mejoras y conclusiones (60–75 s)
Como futura mejora principal, añadiría un sistema de stock para los productos. Esto permitiría mostrar las unidades disponibles, actualizar existencias automáticamente después de cada pedido y avisar cuando un producto esté agotado. Con ello, la tienda sería más útil, más clara para el cliente y más fácil de gestionar para el administrador.

Como conclusión, Agrivall cumple el objetivo de crear una web funcional y responsive, integrando frontend, backend y backoffice. El proyecto tiene una estructura cercana a una aplicación real y deja margen para seguir creciendo, por ejemplo con stock, pagos online, mejoras en reservas o estadísticas de administración.

Muchas gracias.

- Conclusión: proyecto completo, realista y ampliable.
